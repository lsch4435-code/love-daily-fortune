# FORTUNE HOUSE - 오늘의 운세 타로 하우스 🔮

JavaScript로 사용자 입력을 받아, 여러 종류의 운세를 인터랙티브하게 보여주는 웹 페이지입니다.  
 과제 요구사항인 **“사용자 입력 → JS로 처리 → 동적으로 결과 출력”**을 목표로 만들었습니다.

---

## 1. 프로젝트 개요

이 프로젝트는 타로 하우스 느낌의 어두운 분위기 UI에서  
사용자가 이름, 기분, 나이, 별자리, 띠 정보를 입력하면  
각 주제별로 운세를 보여주는 **웹 운세/점집 페이지**입니다.

- HTML/CSS로 페이지 구조와 타로 하우스 분위기를 구현
- JavaScript로 입력값을 읽어 점수를 계산하고, 운세 문장을 선택하여 출력
- 결과는 카드 형태로 애니메이션과 함께 화면에 동적으로 나타납니다.

---

## 2. 주요 기능

### (1) 메인 메뉴 페이지 (`index.html`)

- 프로젝트의 메인 허브 역할
- 다음 다섯 가지 운세 페이지로 이동할 수 있는 메뉴 제공
  - 오늘의 운세 (Daily)
  - 연애 운세 (Love)
  - 재물 운세 (Money)
  - 별자리 운세 (Zodiac)
  - 사주(띠) 운세 (Saju)

---

### (2) 오늘의 운세 - Daily (`daily.html`)

- 입력값
  - 이름
  - 오늘 기분 (행복/설렘/평온/피곤/우울)
  - 나이(선택)
  - 별자리(선택)
  - 태어난 해(띠용, 선택)
- 처리 내용
  - 기분, 별자리, 띠, 연령대 정보를 바탕으로 **오늘의 총 운세 점수(0~100점)**를 계산
  - 점수에 따라 운세 레벨을 세 가지로 분류
    - `상승세` / `무난` / `주의`
  - 운세 점수 + 레벨 + 간단한 설명 문장을 함께 출력
  - 학생·대학생·직장인 등 **연령대별로 한 줄 코멘트**를 다르게 보여줌
- 결과
  - “오늘의 운세 82점 · 상승세 · 대학생/취준” 처럼 점수와 상태를 카드에 표시
  - 아래에 오늘 하루를 위한 짧은 팁/조언 문장을 함께 보여줌

---

### (3) 연애 운세 - Love (`love.html`)

- 입력값은 Daily와 동일 (이름, 기분, 나이, 별자리, 태어난 해)
- 처리 내용
  - 연애 관련 점수 및 운세 문장을 선택
  - 연령대에 따라 포커스가 다르게 설정
    - 학생: 학업·학교 생활 속 연애
    - 대학생/취준: 캠퍼스/알바/스터디 등에서의 인연
    - 직장인: 일과 연애의 균형
  - 기분과 점수에 따라
    - 설레는 상황 / 고백 타이밍 / 조급함을 조심해야 하는 상황 등으로 분기
- 결과
  - 연애 점수, 오늘 연애 흐름, 조심해야 할 점(주의 레벨일 때) + 작은 팁 문장 출력

---

### (4) 재물 운세 - Money (`money.html`)

- 오늘의 돈 흐름과 소비 습관에 대한 운세
- 소비/수입/알바/급여/저축과 관련된 문장들로 구성
- “충동구매 주의”, “작은 수입이 생길 수 있음”, “지출 정리 추천” 등의 메시지 제공
- 나이·연령대에 따라
  - 학생: 용돈, 알바, 공부 투자
  - 사회초년생: 급여·저축·지출 관리
  - 직장인/가족: 가정과 재정 균형
  에 맞는 코멘트를 추가로 보여줌

---

### (5) 별자리 운세 - Zodiac (`zodiac.html`)

- 사용자가 선택한 별자리를 기준으로 운세를 보여주는 페이지
- 별자리를 4원소(불·흙·공기·물)로 묶어서 성향을 반영
  - 예) 불의 별자리: 직진력·열정, 물의 별자리: 감수성·공감
- 오늘 기분과 별자리 기운을 조합해서
  - “오늘은 직진력과 열정이 살아나는 날” 같은 문장을 출력
- 점수가 낮을 경우
  - “충동적인 결정 주의”, “분위기에 휩쓸리지 말 것” 같은 **주의 문장**도 함께 출력

---

### (6) 사주(띠) 운세 - Saju (`saju.html`)

- 태어난 해를 기준으로 **12지 띠(쥐·소·호랑이 ... 돼지)**를 계산
- 각 띠마다
  - 센스, 끈기, 존재감, 활동성 등 **성향 키워드**를 다르게 설정
- 띠의 성향 + 오늘 기분으로 점수를 계산하고
  - “정리하기 좋은 날”, “사람 운이 괜찮은 날”, “휴식이 필요한 안정적인 날” 등의 운세를 출력
- 실제 사주(사주팔자) 전체를 보는 것은 아니고,
  - **“띠 + 오늘 컨디션” 조합으로 간단한 사주 느낌**을 내는 구조입니다.

## 3. 인터랙티브 동작 방식 (JavaScript)

### (1) 입력값 읽기

각 운세 페이지에서 공통으로 다음 요소를 `document.getElementById`로 가져옵니다.

- `nameInput` : 이름
- `moodSelect` : 오늘 기분
- `ageInput` : 나이(선택)
- `zodiacSelect` : 별자리(선택)
- `birthYearInput` : 태어난 해(띠용, 선택)
- `fortuneBtn` : “운세 뽑기” 버튼
- `resultCard`, `resultText`, `resultHint` : 결과 표시 영역

### (2) 점수 계산 로직

1. 기분, 별자리, 띠, 나이(연령대)마다 **각자 다른 가중치(scoreDelta)**를 줌  
   - 예) 행복한 기분일수록 점수 +, 우울할수록 약간 -  
   - 불의 별자리/용띠 → 추진력/존재감 + 점수  
   - 학생/대학생/직장인 등 나이 구간별로 소소한 가중치 부여
2. 기본 점수 50점에서 시작해서,
   - 위 가중치를 모두 합산하여 운세 점수를 계산
3. 마지막에 `Math.random()`을 이용해서
   - -10 ~ +10 사이의 작은 랜덤 값을 더해서  
     → 매번 완전히 같지 않게, **조금씩 달라지는 운세**가 나오도록 구현

### (3) 운세 문장 선택

- 각 카테고리별로 다수의 운세 문장을 배열로 준비해 둠
  - `loveFortunes`, `dailyFortunes`, `moneyFortunes`, `zodiacFortunes`, `sajuFortunes`
- 점수 구간(상승세/무난/주의)에 따라
  - 배열을 3등분해서 그 중 한 구간에서 랜덤으로 문장을 선택
- 선택된 문장에서 `{name}`, `{mood}`, `{sign}`, `{animal}` 같은 플레이스홀더를  
  `.replace()`로 실제 입력값으로 치환해서 출력

### (4) 결과 출력 & 애니메이션

- 버튼 클릭 시 `resultText`, `resultHint`의 `textContent`를 바꿈
- `resultCard`에서 `hidden` 클래스를 제거하고 `show` 클래스를 추가
- CSS에서 `opacity`, `transform`으로 살짝 떠오르는 카드 애니메이션 구현

## 4. 사용 기술

- **HTML5** : 페이지 구조, 입력 폼, 네비게이션
- **CSS3** : 타로 하우스 느낌의 다크 테마, 카드 UI, 반짝이는 배경 연출
- **Vanilla JavaScript** :  
  - DOM 조작 (`getElementById`, `textContent`, `classList`)
  - 조건 분기 및 점수 계산
  - 랜덤 요소를 섞은 운세 문장 선택 로직

## 5. 파일 구조

- `index.html` : 메인 메뉴 페이지
- `daily.html` : 오늘의 전체 운세
- `love.html` : 연애 운세
- `money.html` : 재물 운세
- `zodiac.html` : 별자리 운세
- `saju.html` : 사주(띠) 운세

---

## 6. 실행 방법

1. 이 레포지토리를 클론하거나 ZIP으로 다운로드 후 압축 해제합니다.
2. `index.html` 파일을 브라우저(Chrome, Edge 등)로 열면 됩니다.
3. 메인 메뉴에서 원하는 운세 종류를 클릭하고,
   - 이름, 기분, 나이, 별자리, 태어난 해를 입력한 뒤
   - “운세 뽑기” 버튼을 눌러 결과를 확인합니다.

---

## 7. 한계 및 개선 아이디어

- 현재는 **이름/나이/별자리/띠 + 랜덤**을 섞은 간단한 규칙 기반 운세입니다.
  - 같은 정보를 입력해도 매번 조금씩 다른 결과가 나올 수 있습니다.
- 실제 사주처럼 태어난 시간·장소까지 반영하지는 않습니다.
- 추후에는
  - 이름/생년월일 기반으로 같은 사람에게는 하루에 하나의 고정된 결과만 보여주기
  - 더 많은 운세 문장과 카테고리(우정운, 시험운 등) 추가
  - LocalStorage를 사용하여 최근에 본 운세 기록하기
  같은 방향으로 확장해 볼 수 있습니다.
